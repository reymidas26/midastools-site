## Talk Outlines & Structure

### Prompt 1 — Conference Talk Outline Generator

Use this to create a complete talk outline for any conference presentation, from a 15-minute breakout to a 45-minute keynote.

```
You are an elite speaking coach who has helped prepare speakers for TED, SXSW, Web Summit, and Fortune conferences. Create a conference talk outline.

Talk title: [YOUR TALK TITLE — or describe the topic and I'll suggest titles]
Conference/event: [CONFERENCE NAME AND TYPE — e.g., "SaaStr Annual, B2B SaaS conference"]
Audience: [WHO WILL BE THERE — e.g., "500 startup founders and VCs," "200 enterprise marketers," "50 CIOs"]
Talk length: [15 MIN / 20 MIN / 30 MIN / 45 MIN]
My expertise/credential: [WHY I'M QUALIFIED TO GIVE THIS TALK]
Core message: [THE ONE THING THE AUDIENCE SHOULD REMEMBER — 1 SENTENCE]
Key takeaways: [LIST 3-5 THINGS THE AUDIENCE SHOULD LEARN OR BE ABLE TO DO]
My speaking style: [e.g., "data-driven," "storytelling-focused," "provocative," "practical/tactical"]

Generate:

1. **Talk Title Options** — 3 compelling title options (if not already finalized):
   - A curiosity-driven title
   - A bold claim title
   - A practical/tactical title

2. **Complete Outline** — Section-by-section outline with:
   - Section name
   - Time allocation
   - Key points to cover
   - Transition to next section
   - Audience engagement moment (question, exercise, or surprise)

3. **Opening (First 90 Seconds)** — Three options:
   - Option A: Story-led opening
   - Option B: Data/statistic-led opening
   - Option C: Provocative question opening
   - For each: exact words to say, what to show on screen, body language guidance

4. **Core Content Structure** — The main body organized using one of these frameworks (pick the best fit):
   - Problem → Insight → Solution → Proof → Action
   - Past → Present → Future
   - Myth → Reality → Opportunity
   - Challenge 1 → Challenge 2 → Challenge 3 → Unified Solution

5. **Closing (Last 2 Minutes)** — A memorable ending:
   - Callback to the opening
   - The one thing to remember
   - Specific call to action for the audience
   - Final statement (designed to be quotable/tweetable)

6. **Slide Recommendations** — Suggested number of slides and what each should contain (visual-heavy, minimal text)

7. **Timing Breakdown** — Minute-by-minute timeline for the entire talk

This outline should make the speaker feel prepared and the audience feel transformed.
```

### Prompt 2 — TED-Style Talk Script

Use this to write a full script for a short, high-impact talk in the TED format (under 18 minutes).

```
You are a TED talk scriptwriter. Write a full script for a TED-style talk.

Topic: [YOUR TOPIC]
Speaker: [YOUR NAME AND BRIEF BIO]
Core idea: [THE ONE IDEA WORTH SPREADING — 1 SENTENCE]
Target length: [8 MIN / 12 MIN / 15 MIN / 18 MIN]
Audience: [GENERAL / TECHNICAL / BUSINESS / MIXED]

Personal stories I can reference:
- [STORY 1 — a personal experience related to the topic]
- [STORY 2 — another experience or anecdote]
- [STORY 3 — optional]

Data/research I want to include:
- [DATA POINT 1]
- [DATA POINT 2]
- [DATA POINT 3]

The audience currently believes: [CONVENTIONAL WISDOM YOU'RE CHALLENGING]
After my talk, they should believe: [THE NEW PERSPECTIVE]

Generate:

1. **Full Script** — A complete, word-for-word script written for speaking (not reading):
   - Conversational, not academic
   - Short sentences and paragraphs
   - Natural pauses marked with [PAUSE]
   - Emphasis words marked with *emphasis*
   - Audience reaction moments marked with [LAUGHTER EXPECTED] or [AUDIENCE REFLECTION]

2. **Structure Breakdown** — Mark each section of the script:
   - Hook (first 30 seconds)
   - Context/setup (why this matters)
   - The insight/idea
   - Supporting evidence (stories + data)
   - Implications (so what?)
   - Call to action (what to do with this idea)

3. **Stage Directions** — At key moments, note:
   - Where to move on stage
   - When to gesture
   - When to slow down vs. speed up
   - When to make eye contact with specific areas of the audience

4. **Slide Cues** — Mark where slides should advance and what each slide should show (minimal text — mostly images, data visualizations, or single words)

5. **Quotable Moments** — Identify 3 lines in the script designed to be shared on social media

6. **Word Count** — Confirm the script matches the target length (approximately 130 words per minute for speaking pace)

The script should feel authentic to the speaker's voice, not generic. It should have moments of vulnerability, humor, and insight.
```

### Prompt 3 — Technical Talk Outline

Use this for developer conferences, engineering meetups, or any technical presentation where the audience expects depth and code-level insight.

```
You are a developer advocate who gives technical talks at conferences like re:Invent, KubeCon, Strange Loop, and PyCon. Create a technical talk outline.

Talk topic: [TECHNICAL TOPIC — e.g., "Building real-time ML pipelines with Kafka and Python"]
Conference: [CONFERENCE NAME]
Audience technical level: [BEGINNER / INTERMEDIATE / ADVANCED / MIXED]
Talk length: [20 MIN / 30 MIN / 45 MIN]
Technologies covered: [LIST SPECIFIC TECHNOLOGIES, FRAMEWORKS, LANGUAGES]
My role/expertise: [YOUR BACKGROUND]

What the audience should learn:
- [LEARNING OUTCOME 1]
- [LEARNING OUTCOME 2]
- [LEARNING OUTCOME 3]

Demo/code I can show: [DESCRIBE ANY LIVE CODING, DEMOS, OR CODE SAMPLES YOU HAVE]
Architecture I can diagram: [DESCRIBE ANY SYSTEM ARCHITECTURE TO PRESENT]

Generate:

1. **Talk Title Options** — 3 options:
   - Technical + descriptive
   - Problem-focused
   - Lessons-learned format ("What I Learned Building X at Scale")

2. **Complete Outline:**

   **Opening (3-5 min)**
   - The problem this tech solves (not the tech itself)
   - A relatable "war story" or production incident that motivates the talk
   - "Here's what you'll walk away with today"

   **Context/Background (5-8 min)**
   - Landscape of approaches (what are the options?)
   - Why existing approaches fall short
   - The architectural decision and trade-offs

   **Deep Dive (12-20 min)**
   - Core concept 1: explanation + code/diagram
   - Core concept 2: explanation + code/diagram
   - Core concept 3: explanation + code/diagram
   - For each: common pitfalls and how to avoid them
   - Live demo or walkthrough markers

   **Lessons Learned (5-8 min)**
   - 3-5 hard-won lessons from production experience
   - What went wrong and how you fixed it
   - Performance numbers (before/after)

   **Closing (2-3 min)**
   - Summary of key takeaways
   - Resources to go deeper (GitHub repos, docs, blog posts)
   - How to reach you for questions

3. **Slide Design Guidance**
   - When to use code on slides (and how to format it for readability)
   - When to use architecture diagrams vs. flow charts
   - Font sizes and syntax highlighting recommendations
   - How many lines of code per slide (max)

4. **Demo Script** — If doing a live demo:
   - What to have pre-built vs. live-code
   - Backup plan if the demo fails
   - Key moments to call out what's happening

5. **Q&A Prep** — 10 likely technical questions and prepared answers

Make the outline technically rigorous but accessible. Every concept should have both an explanation and a concrete example.
```

### Prompt 4 — Thought Leadership / Industry Keynote Outline

Use this for a keynote address where you're presenting a vision or perspective about the future of your industry.

```
You are a keynote speechwriter who has written for industry leaders and Fortune 500 CEOs. Create a keynote talk outline.

Speaker: [YOUR NAME, TITLE, COMPANY]
Event: [EVENT NAME AND CONTEXT]
Audience: [SIZE AND COMPOSITION — e.g., "2,000 fintech professionals"]
Talk length: [30 MIN / 45 MIN / 60 MIN]
Industry: [YOUR INDUSTRY]

Your thesis: [YOUR MAIN ARGUMENT ABOUT WHERE THE INDUSTRY IS HEADING — 2-3 SENTENCES]
Supporting trends: [LIST 3-4 TRENDS THAT SUPPORT YOUR THESIS]
Contrarian element: [WHERE YOUR VIEW DIFFERS FROM CONVENTIONAL WISDOM]
Your company's role: [HOW YOUR COMPANY FITS INTO THIS VISION — optional, can be subtle]

Generate:

1. **Keynote Title Options** — 3 titles that feel big and important:
   - Visionary: "The [X] Era: What Comes Next for [INDUSTRY]"
   - Provocative: "Everything We Think We Know About [TOPIC] Is Wrong"
   - Practical: "[NUMBER] Shifts That Will Define [INDUSTRY] in the Next Decade"

2. **Opening Sequence (5 minutes)**
   - Attention-grabbing opening (a story, a bold statement, a visual)
   - Why this topic, why now, why it matters to everyone in the room
   - Preview of the talk's structure

3. **Act 1: The Current Reality (10 minutes)**
   - Where the industry is today
   - The forces creating change
   - Data points and examples that illustrate the shift
   - A story of a company or person experiencing this change firsthand

4. **Act 2: The Shift (12-15 minutes)**
   - Your thesis: what's changing and why
   - Trend 1: evidence + example + implication
   - Trend 2: evidence + example + implication
   - Trend 3: evidence + example + implication
   - The contrarian insight: why most people are wrong about [ASPECT]

5. **Act 3: The Future (8-10 minutes)**
   - What the industry looks like in 3-5 years if your thesis is correct
   - Winners vs. losers: what separates companies that thrive from those that struggle
   - Specific opportunities for the audience to act on

6. **Closing (3-5 minutes)**
   - Personal reflection or story that brings it home
   - The call to action: what every person in the room should do this week
   - A memorable closing line

7. **Slide Strategy**
   - Key visuals (not bullet points) for each section
   - Data visualizations to support major claims
   - Moment-of-impact slides (single word or image)

8. **Quotable Lines** — 5 lines throughout the talk designed for social media and press coverage

A great keynote makes the audience feel like they just saw the future. This outline should create that experience.
```

## Opening Hooks & Storytelling

### Prompt 5 — Opening Hook Generator

Use this when you need a powerful opening for any presentation. The first 60 seconds determine whether the audience pays attention.

```
You are a presentation opening specialist. Create 7 different opening hooks for a presentation.

Topic: [YOUR PRESENTATION TOPIC]
Audience: [WHO — THEIR ROLE, INDUSTRY, CONTEXT]
Talk length: [TOTAL TALK LENGTH]
Tone: [SERIOUS / INSPIRATIONAL / FUNNY / PROVOCATIVE / CONVERSATIONAL]
Context: [WHERE YOU'RE PRESENTING — e.g., "conference main stage," "board room," "webinar"]

Generate 7 opening options, each with the exact words to say (30-60 seconds each):

1. **The Shocking Statistic**
   - A surprising data point that reframes the audience's understanding
   - Why it matters to them specifically
   - Bridge to the rest of the talk

2. **The Personal Story**
   - A brief personal anecdote that illustrates the core theme
   - Vulnerability or humor (or both)
   - Clear connection to the audience's experience

3. **The Provocative Question**
   - A question that makes the audience genuinely pause and think
   - Not rhetorical — actually invite mental engagement
   - Follow up that reveals why the question matters

4. **The Bold Prediction**
   - A specific, testable claim about the future
   - "Within [TIMEFRAME], [PREDICTION]. And here's why..."
   - Should feel slightly uncomfortable but plausible

5. **The Contrarian Statement**
   - Challenge a widely held belief in the audience's world
   - "Everyone in this room believes [X]. I'm going to show you why that's wrong."
   - Immediately promise the evidence to back it up

6. **The "Imagine" Scenario**
   - Paint a vivid picture of a future state or hypothetical
   - Engage the senses: what would the audience see, feel, experience
   - Pull them back to reality: "That future is closer than you think"

7. **The Audience Participation Opening**
   - A quick show-of-hands or ask-your-neighbor exercise
   - Uses the audience's own response to set up the theme
   - Works best for in-person and live virtual events

For each option, include:
- The exact opening words (scripted)
- The transition sentence to the rest of the talk
- What to show on the screen (if anything)
- Body language and delivery notes
- Which talk types this opening works best for

Also provide guidance on which opening to choose based on the audience and context.
```

### Prompt 6 — Storytelling Framework for Presentations

Use this to structure stories within your presentation that are emotionally engaging and strategically relevant.

```
You are a storytelling expert who helps speakers weave narratives into professional presentations. Help me craft stories for my talk.

Talk topic: [YOUR TOPIC]
Core message: [THE ONE THING YOU WANT THE AUDIENCE TO REMEMBER]
Audience: [WHO]

Stories I have to work with:
- Story A: [BRIEF DESCRIPTION — e.g., "When my startup almost ran out of money and we had to pivot in 48 hours"]
- Story B: [BRIEF DESCRIPTION — e.g., "A customer who used our product in a way we never expected"]
- Story C: [BRIEF DESCRIPTION — e.g., "A lesson from my first job that I still use today"]
(Provide at least 2 stories, or describe experiences and I'll help shape them)

Generate:

1. **Story Selection** — Which stories to use, where in the talk, and why:
   - Story placement map (opening, middle, closing)
   - How each story supports the core message
   - Which story creates the emotional peak

2. **Story Structures** — For each selected story, a fully developed narrative:
   - **Setup** (15-20 seconds): Set the scene — when, where, who, what was at stake
   - **Tension** (20-30 seconds): The conflict, challenge, or unexpected twist
   - **Resolution** (15-20 seconds): What happened, what changed
   - **Lesson** (10 seconds): The takeaway, connected explicitly to the audience's world
   - **Transition** (5 seconds): Bridge to the next section of the talk

3. **Sensory Details** — For each story, 3-4 specific details that make it vivid:
   - A visual detail (what something looked like)
   - A temporal detail (time of day, date, how long)
   - An emotional detail (what you felt physically)
   - A dialogue snippet (exact words someone said)

4. **Story Don'ts** — For each story, what to leave out:
   - Details that don't serve the point
   - Tangents that could derail the narrative
   - How long each story should take (most speakers go too long)

5. **The Meta-Narrative** — How all stories connect to form a single narrative thread through the entire talk

6. **Vulnerability Guide** — How much personal vulnerability to show:
   - What level of openness is appropriate for this audience
   - How to be vulnerable without being unprofessional
   - The difference between vulnerability and oversharing

Stories should feel natural, not forced. The audience should think "that was a great talk" not "that was a story-heavy talk."
```

### Prompt 7 — Talk Narrative Arc Builder

Use this to ensure your entire presentation has a compelling narrative structure, not just a series of slides.

```
You are a narrative architect who designs the emotional and intellectual journey of presentations. Build a narrative arc for my talk.

Talk topic: [YOUR TOPIC]
Talk length: [DURATION]
Audience: [WHO AND WHAT THEY CARE ABOUT]
Current structure: [DESCRIBE YOUR CURRENT SLIDE ORDER OR OUTLINE — even if rough]

What the audience knows coming in: [THEIR EXISTING KNOWLEDGE/BELIEFS ABOUT YOUR TOPIC]
What you want them to feel by the end: [THE EMOTIONAL STATE — e.g., "inspired to act," "alarmed enough to change," "equipped with new tools"]
What you want them to do after: [THE SPECIFIC ACTION — e.g., "try this technique," "rethink their strategy," "buy our product"]

Generate:

1. **Narrative Arc Diagram** — Describe the emotional arc of your talk as a journey:
   - Starting emotional state → key emotional beats → ending emotional state
   - Map it to the classic story structure: setup → rising action → climax → falling action → resolution

2. **Tension and Release Points** — Identify 3-4 moments of tension (things that make the audience uncomfortable, curious, or uncertain) and how each is resolved

3. **The "Turn"** — The pivotal moment in the talk where the audience's perspective shifts:
   - When it happens (minute mark)
   - What you say/show that causes the shift
   - Why it's earned (what you've built up to make this moment land)

4. **Pacing Guide** — When to:
   - Speed up (energy, excitement)
   - Slow down (importance, reflection)
   - Pause (let something land)
   - Use humor (release tension)
   - Show data (credibility)
   - Tell stories (connection)

5. **Audience Questions** — At each stage of the talk, what the audience is thinking:
   - Opening: "Why should I care?"
   - Middle: "How does this work?" / "Is this true?"
   - Climax: "Oh, I didn't see it that way"
   - Close: "What should I do about this?"

6. **Connective Tissue** — Transition sentences that tie each section to the next, maintaining the narrative thread so nothing feels disconnected

7. **The Promise and Payoff** — What you promise in the opening and how you deliver on it by the closing

This framework should transform a collection of slides into a story that the audience lives through.
```

## Panel Preparation & Moderation

### Prompt 8 — Panel Preparation Guide

Use this when you're participating on a panel discussion and need to prepare your key talking points and memorable moments.

```
You are a media trainer who prepares executives for high-profile panel discussions. Prepare me for a panel.

Panel topic: [TOPIC OF THE PANEL]
Event: [EVENT NAME]
Panel title: [EXACT TITLE IF KNOWN]
Other panelists: [LIST NAMES AND THEIR COMPANIES/ROLES — OR "UNKNOWN"]
Moderator: [NAME — OR "UNKNOWN"]
Audience: [WHO AND HOW MANY]
Duration: [LENGTH]
Format: [MODERATED Q&A / STRUCTURED DISCUSSION / FIRESIDE CHAT]

My role/company: [YOUR TITLE AND COMPANY]
My perspective on this topic: [YOUR MAIN VIEWPOINT IN 2-3 SENTENCES]
What I want to promote (subtly): [e.g., "our new product," "our expertise in X," "our company as a thought leader"]
My worry: [WHAT I'M MOST CONCERNED ABOUT — e.g., "going blank," "being overshadowed," "getting a hostile question"]

Generate:

1. **3 Key Messages** — Three points I want to make regardless of what questions are asked:
   - Message 1: [With a supporting data point or story]
   - Message 2: [With a supporting data point or story]
   - Message 3: [With a supporting data point or story]
   - For each: a "bridge phrase" to steer any question toward this message

2. **Predicted Questions** — 10 questions the moderator might ask, with:
   - The question
   - My prepared response (30-45 seconds — panels are about brevity)
   - A way to end each answer that invites follow-up or positions me favorably

3. **Sound Bites** — 5 quotable statements I can work into the conversation:
   - Each under 15 words
   - Memorable, specific, not generic
   - Designed to be tweeted or quoted in event coverage

4. **Differentiation Strategy** — How to stand out from other panelists:
   - If others go tactical, I go strategic (or vice versa)
   - A contrarian or unique angle on the topic
   - How to respectfully disagree to create interesting discussion

5. **Body Language on Panels**
   - How to sit (posture, hands, eye contact)
   - How to listen actively while others talk
   - How to signal you want to speak without interrupting
   - What to do when someone says something you disagree with

6. **Opening and Closing Moves**
   - If asked to introduce yourself: a 15-second intro that's different from a standard bio
   - If given closing remarks: a 20-second close that people remember

7. **Worst-Case Prep**
   - If asked something you don't know: how to handle it gracefully
   - If another panelist is dominating: how to reclaim space
   - If the moderator is poor: how to self-moderate
   - If you make a mistake: recovery framework

Panels are about personality and presence, not perfect answers. Prepare me to be the panelist people approach afterward.
```

### Prompt 9 — Panel Moderator Preparation Guide

Use this when you're moderating a panel and need to prepare questions, manage dynamics, and create a great audience experience.

```
You are an experienced panel moderator who has moderated discussions at Davos, TED, and major industry conferences. Help me prepare to moderate a panel.

Panel topic: [TOPIC]
Event: [EVENT NAME]
Panelists:
- [NAME, TITLE, COMPANY — their known perspective/expertise]
- [NAME, TITLE, COMPANY — their known perspective/expertise]
- [NAME, TITLE, COMPANY — their known perspective/expertise]
- [NAME, TITLE, COMPANY — their known perspective/expertise] (optional)
Audience: [WHO AND SIZE]
Duration: [LENGTH]
Format preferences: [ANY FORMAT REQUIREMENTS FROM THE EVENT ORGANIZER]
Audience Q&A: [YES — HOW MUCH TIME / NO]

Generate:

1. **Opening (3 minutes)**
   - How to set the stage and frame the conversation
   - A brief, energizing hook that makes the audience care about this topic right now
   - How to introduce panelists (not reading bios — a more engaging format)

2. **Question Flow** — 12-15 questions organized in three acts:
   - Act 1 (warmup): Accessible questions that get everyone talking
   - Act 2 (depth): Challenging questions that create debate and insight
   - Act 3 (future/action): Forward-looking questions that leave the audience with takeaways
   - For each question: who to direct it to first, and a follow-up prompt

3. **Debate Catalysts** — 3 questions specifically designed to create productive disagreement:
   - A "which is better: A or B?" format
   - A "what's the biggest myth?" format
   - A "where are we all wrong?" format

4. **Traffic Management Guide**
   - How to redirect a long-winded panelist (exact phrases)
   - How to draw out a quiet panelist
   - How to manage a panelist who keeps pitching their company
   - How to disagree with a panelist as a moderator (if needed)

5. **Audience Q&A Management**
   - How to transition to audience questions
   - How to handle bad audience questions (rambling, off-topic, statements disguised as questions)
   - How to field questions to the right panelist

6. **Closing (2 minutes)**
   - A "lightning round" option: one sentence from each panelist on [SPECIFIC PROMPT]
   - How to wrap up and give the audience a clear takeaway
   - How to thank panelists and close

7. **Timing Plan** — Minute-by-minute breakdown for the entire panel

The moderator's job is to create the best conversation the audience has ever heard on this topic. Make me that moderator.
```

## Audience Engagement & Interaction

### Prompt 10 — Audience Engagement Strategy

Use this to plan interactive moments throughout your presentation that keep the audience involved and energized.

```
You are an audience engagement specialist who designs interactive experiences for conference presentations. Create an engagement strategy for my talk.

Talk topic: [YOUR TOPIC]
Talk length: [DURATION]
Audience size: [NUMBER]
Audience type: [EXECUTIVES / DEVELOPERS / MARKETERS / MIXED / STUDENTS]
Format: [IN-PERSON / VIRTUAL / HYBRID]
Tech available: [e.g., "Slido for polls," "no tech — just me and a mic," "Zoom with chat"]
My comfort level with interaction: [HIGH — I LOVE IMPROV / MEDIUM — STRUCTURED IS FINE / LOW — MINIMAL INTERACTION PREFERRED]

Generate:

1. **Engagement Map** — Place 5-7 interactive moments throughout the talk:
   - Minute mark
   - Type of interaction
   - Purpose (energy, reflection, data gathering, humor, connection)
   - Transition into and out of the interaction

2. **Opening Interaction** — Get the audience engaged in the first 2 minutes:
   - For in-person: a show of hands, stand up/sit down, or turn-to-your-neighbor exercise
   - For virtual: a poll, chat prompt, or reaction exercise
   - How to use the audience's response to customize your opening

3. **Mid-Talk Energy Boost** — An activity for the energy dip (usually at the 40-60% mark):
   - A quick exercise, thought experiment, or audience challenge
   - Takes 60-90 seconds
   - Re-energizes without derailing the content

4. **Audience Polling Strategy** — 2-3 poll questions to ask during the talk:
   - The question
   - Answer options
   - What each result tells you (and how to react to it live)
   - How to use the poll results to make a point

5. **Discussion Prompts** — If the format allows small group discussion:
   - 2-3 prompts for turn-and-talk or table discussions
   - Time allocation (2-3 minutes each)
   - Debrief strategy (how to harvest insights from the room)

6. **Q&A Strategy** — How to make Q&A valuable, not awkward:
   - When to do Q&A (end, throughout, or both)
   - How to seed the first question (if the audience is shy)
   - How to handle challenging questions
   - How to redirect off-topic questions

7. **Closing Interaction** — End with the audience doing something:
   - A commitment exercise ("Turn to your neighbor and share one thing you'll do differently")
   - A group response ("If you're committed to [THING], stand up / raise your hand")
   - A reflective prompt for individual application

8. **Backup Plans** — For each interaction, what to do if:
   - No one participates
   - Too many people want to participate
   - The technology fails
   - The interaction takes longer than planned

The engagement strategy should feel organic, not gimmicky. Every interaction should serve the content, not distract from it.
```

### Prompt 11 — Workshop / Interactive Session Design

Use this when your conference session is a hands-on workshop, not a lecture-style talk.

```
You are a workshop facilitator who designs interactive learning sessions for conferences. Create a workshop plan.

Workshop topic: [TOPIC]
Event: [EVENT NAME]
Duration: [60 MIN / 90 MIN / HALF DAY / FULL DAY]
Participants: [NUMBER AND TYPE — e.g., "30 product managers," "50 startup founders"]
Learning objectives:
- [OBJECTIVE 1 — what they should be able to do after]
- [OBJECTIVE 2]
- [OBJECTIVE 3]
Materials available: [e.g., "whiteboard," "laptops," "sticky notes," "printed worksheets," "none"]
Room setup: [THEATER / CLASSROOM / U-SHAPE / TABLES OF 4-6 / STANDING]

Generate:

1. **Workshop Flow** — Complete minute-by-minute agenda:
   - Time | Activity | Format (lecture/individual/pairs/groups) | Purpose
   - Balance between instruction and hands-on practice (aim for 30/70 split)
   - Built-in breaks for sessions over 60 minutes

2. **Opening Activity (10 minutes)**
   - Icebreaker that's relevant to the topic (not generic)
   - Sets the stage for the learning ahead
   - Helps participants get comfortable with each other

3. **Core Content Blocks** — For each learning objective:
   - Brief instruction (5-10 minutes max)
   - Guided exercise (15-20 minutes)
   - Exercise instructions (what to do, step by step)
   - Expected output (what they should produce)
   - Debrief questions (how to discuss the exercise as a group)

4. **Group Exercises** — 2-3 collaborative activities:
   - Clear instructions (written, not just verbal)
   - Group size and formation method
   - Time limit and milestones ("In 5 minutes, you should have X")
   - Presentation/share-back format

5. **Worksheet/Template Content** — If using worksheets:
   - Content for each worksheet/template
   - Instructions printed on the worksheet
   - Examples pre-filled for reference

6. **Facilitation Guide** — How to manage the room:
   - How to give clear instructions for exercises
   - How to time-check without being annoying
   - How to handle groups that finish early or fall behind
   - How to manage dominant participants
   - How to encourage quiet participants

7. **Closing Activity (10 minutes)**
   - Reflection exercise: "What's your #1 takeaway?"
   - Action planning: "What will you do in the next 7 days?"
   - Share-out method
   - Resource list for continued learning

8. **Facilitator Preparation Checklist**
   - Materials to prepare
   - Room setup requirements
   - Technology to test
   - Backup plans for each section

A great workshop leaves participants with something they built, not just something they heard. Design this workshop around doing, not listening.
```

## Specific Presentation Formats

### Prompt 12 — Lightning Talk / Ignite Format (5-Minute Presentation)

Use this for lightning talks, Ignite presentations (20 slides, 15 seconds each), or any very short format.

   - Morning routine
   - What to eat and drink (and what to avoid)
   - Vocal warm-up
   - Mental preparation technique (visualization)
   - 15-minute warm-up run-through strategy (not a full rehearsal)
   - Arrival and setup checklist

6. **Confidence Builders**
   - How to mentally frame nervousness as excitement
   - A pre-stage ritual to get in the zone
   - What to do in the 60 seconds before you start speaking
