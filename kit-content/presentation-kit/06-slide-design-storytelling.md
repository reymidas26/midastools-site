# 06 — Slide Design & Storytelling Prompts

## 20+ AI Prompts for Presentation Design and Narrative Craft

These prompts help you optimize slide copy, structure visual hierarchy, describe data visualizations, build narrative arcs, and customize presentations for specific audiences. Use these prompts to polish any presentation — regardless of the topic — into something memorable and professional.

**Works with:** ChatGPT, Claude, Gemini, or any LLM.

## Slide Copy Optimization

### Prompt 1 — Slide Copy Editor and Optimizer

Use this to take your existing slide text and make it sharper, clearer, and more impactful.

```
You are a presentation copy editor who specializes in making slide content concise and impactful. Optimize my slide copy.

Here is my current slide content:

[PASTE YOUR SLIDE TEXT HERE — INCLUDE SLIDE TITLE AND ALL BULLET POINTS/TEXT]

Context:
- Presentation type: [PITCH DECK / SALES DECK / KEYNOTE / INTERNAL REPORT / TRAINING]
- Audience: [WHO]
- This slide's purpose: [WHAT THIS SLIDE SHOULD ACCOMPLISH — e.g., "convince investors the market is large," "make the audience laugh," "explain the product simply"]
- Tone: [FORMAL / CONVERSATIONAL / INSPIRING / DATA-DRIVEN]

For each slide, provide:

1. **Current Assessment**
   - What's working
   - What's not working
   - Word count (current vs. recommended)

2. **Optimized Version**
   - Rewritten title (as an insight headline, not a label)
   - Rewritten body content (concise, slide-appropriate)
   - Maximum 5-7 words per bullet point
   - No complete sentences on slides (save those for speaker notes)
   - Active voice throughout

3. **Speaker Notes Version**
   - The full talking track to go with the optimized slide
   - All the context, data, and narrative that should be SPOKEN, not displayed

4. **Alternative Titles** — 3 title options:
   - Option A: Data-driven ("40% of teams report...")
   - Option B: Benefit-driven ("Cut your review time in half")
   - Option C: Question-driven ("What if reporting took 5 minutes?")

5. **Visual Recommendation** — How this slide should look:
   - Text layout (centered, left-aligned, split layout)
   - Whether to add an image, icon, chart, or keep text-only
   - Color emphasis suggestions

Rules:
- No more than 6 bullet points per slide
- No more than 6 words per bullet point
- Headlines should tell the insight, not describe the slide
- Every word must earn its place — cut ruthlessly
```

### Prompt 3 — Bullet Point Transformer

Use this to transform dense paragraph text or long bullets into clean, scannable slide content.

```
You are a slide content specialist. Transform this text into slide-ready format.

Here is my current content (it's too dense for a slide):

[PASTE PARAGRAPH TEXT, LONG BULLET POINTS, OR ESSAY-STYLE CONTENT]

This content needs to become: [1 SLIDE / 2 SLIDES / 3 SLIDES]
Audience: [WHO]
Key message: [WHAT'S THE MAIN TAKEAWAY FROM THIS CONTENT]

Generate:

1. **Slide-Ready Version** — Restructured for slide format:
   - Headline (insight-driven)
   - 3-5 bullet points (max 7 words each)
   - One key number or callout highlighted
   - Zero paragraphs on the slide

2. **Information Hierarchy** — Rank every piece of information:
   - Must include (essential to the message)
   - Nice to have (supports but not critical)
   - Cut (move to speaker notes or appendix)

3. **Speaker Notes** — All the context and detail that was cut from the slide:
   - Full talking track with the rich detail
   - Transitions and explanations
   - Data sources and supporting evidence

4. **Visual Alternatives** — 2 ways to show this information without text:
   - Option A: Chart or diagram description
   - Option B: Icon-based or visual metaphor

The slide should be understandable in 3 seconds of scanning. If it takes more than 3 seconds to read, it's too dense.
```

## Visual Hierarchy & Slide Layout

### Prompt 4 — Slide Layout Advisor

Use this to get recommendations on how to visually arrange content on specific slides for maximum impact.

```
You are a presentation designer who creates visually compelling slide layouts. Advise on layout for my slides.

Presentation tool: [POWERPOINT / GOOGLE SLIDES / KEYNOTE / FIGMA / CANVA]
Style: [MINIMAL / CORPORATE / BOLD / CREATIVE / DATA-HEAVY]
Color palette: [DESCRIBE — e.g., "navy blue and white," "brand colors are green and gray," "dark background preferred"]

For each slide below, recommend a layout:

Slide 1:
- Content type: [e.g., "title slide," "data comparison," "customer testimonials," "team bios," "timeline"]
- Content: [DESCRIBE WHAT'S ON THE SLIDE]
- Number of elements: [e.g., "3 bullet points + 1 image," "4 data points," "6 team photos"]

Slide 2:
- Content type: [DESCRIBE]
- Content: [DESCRIBE]
- Number of elements: [COUNT]

[Continue for all slides needing layout advice]

For each slide, provide:

1. **Layout Recommendation**
   - Layout type: full-bleed, split (50/50 or 60/40), centered, grid, asymmetric
   - Content placement: where each element goes
   - White space strategy: what to leave empty
   - Visual hierarchy: what the eye should see first, second, third

2. **Typography Guidance**
   - Title: font size range, weight (bold/light)
   - Body text: font size, alignment
   - Callout numbers: how to make data points pop
   - Maximum number of font sizes per slide (recommend 2-3)

3. **Visual Elements**
   - Icon recommendations (what to depict)
   - Image style (photo, illustration, abstract)
   - Chart type (if data is present)
   - Color usage: primary, accent, background

4. **Common Mistakes to Avoid**
   - For this layout type, what usually goes wrong
   - How to prevent it

5. **Accessibility Check**
   - Font size minimum (18pt for body, 24pt+ for titles)
   - Color contrast recommendations
   - Alternative for color-blind viewers

The goal is a slide that communicates its message through visual structure, not just words.
```

### Prompt 5 — Presentation Theme and Style Guide Creator

Use this to establish a consistent visual language for your entire presentation.

```
You are a brand presentation designer. Create a presentation style guide.

Context:
- Company/brand: [COMPANY NAME]
- Brand colors: [LIST HEX CODES OR DESCRIBE — e.g., "#1a2b3c navy, #ff6600 orange, #ffffff white"]
- Brand fonts: [IF KNOWN — e.g., "Inter for headers, Source Sans for body"]
- Logo: [DESCRIBE — e.g., "simple wordmark, dark version on light backgrounds"]
- Industry: [INDUSTRY]
- Presentation purpose: [PITCH DECK / SALES / KEYNOTE / INTERNAL]
- Vibe: [e.g., "modern and clean," "bold and energetic," "sophisticated and minimal," "friendly and approachable"]

Generate a presentation style guide:

1. **Color System**
   - Primary color: usage rules (backgrounds, headers, accents)
   - Secondary color: usage rules
   - Accent color: for highlights, callouts, data emphasis
   - Background options: light, dark, gradient recommendations
   - Text colors: on light backgrounds, on dark backgrounds
   - Data visualization colors: a palette of 4-6 chart colors that work together

2. **Typography System**
   - Title font: name, weight, size range, case (sentence vs. title)
   - Subtitle font: name, weight, size
   - Body text: name, weight, size, line spacing
   - Data callout: how to style large numbers or statistics
   - Consistency rules: never more than 2 font families, 3 sizes

3. **Slide Templates** — Describe the layout for each type:
   - Title slide
   - Section divider slide
   - Content slide (text-heavy)
   - Data/chart slide
   - Image + text slide
   - Quote/testimonial slide
   - Comparison slide
   - Timeline slide
   - Team/people slide
   - Closing/CTA slide

4. **Image & Icon Guidelines**
   - Photo style: professional, candid, abstract, illustration
   - Icon style: line, filled, flat, consistent set recommendation
   - Image treatment: rounded corners, shadows, full-bleed, framed
   - Logo placement on each slide type

5. **Do / Don't Rules**
   - Do: use consistent spacing, align elements to a grid, use white space
   - Don't: use clip art, center everything, use more than 3 colors per slide
   - Do: animate subtly (fade in) or not at all
   - Don't: use bouncing, spinning, or flying animations

6. **Slide Transition Recommendations**
   - Between sections: what transition to use
   - Within sections: no transition (instant)
   - Animation for data reveals: fade or wipe (sparingly)

This style guide should make it possible for anyone on the team to create a slide that looks like it belongs in the same deck.
```

### Prompt 6 — Slide Deck Consistency Audit

Use this to review an existing presentation for consistency issues in design, messaging, and flow.

```
You are a presentation quality reviewer. Audit my presentation for consistency and quality issues.

I'll describe my current deck slide by slide:

Slide 1: [TITLE — CONTENT SUMMARY — LAYOUT DESCRIPTION]
Slide 2: [TITLE — CONTENT SUMMARY — LAYOUT DESCRIPTION]
Slide 3: [TITLE — CONTENT SUMMARY — LAYOUT DESCRIPTION]
[Continue for all slides]

Presentation details:
- Total slides: [NUMBER]
- Target length: [MINUTES]
- Audience: [WHO]
- Purpose: [WHAT]

Audit the following:

1. **Content Consistency**
   - Are key terms used consistently throughout?
   - Are claims made early supported by evidence later?
   - Are there contradictions between slides?
   - Is the level of detail consistent?

2. **Narrative Flow**
   - Does the story build logically from slide to slide?
   - Are there any gaps where the audience will be confused?
   - Are transitions clear?
   - Does the conclusion naturally follow from the content?

3. **Visual Consistency**
   - Are layouts consistent for similar slide types?
   - Is the title style consistent (all caps, sentence case, etc.)?
   - Are bullet point styles consistent (punctuation, capitalization)?
   - Are chart styles consistent?

4. **Pacing Issues**
   - Slides that are too dense (need splitting)
   - Slides that are too thin (need combining)
   - Sections that are too long or too short
   - Energy dips (where the audience might disengage)

5. **Missing Elements**
   - Slides that should be added
   - Content that's promised but never delivered
   - A call to action (if missing)
   - Speaker credibility establishment (if missing)

6. **Priority Fixes** — Ordered list of the 5 most impactful improvements:
   - Fix 1 (highest impact)
   - Fix 2
   - Fix 3
   - Fix 4
   - Fix 5

Be specific and actionable in every recommendation.
```

## Data Visualization Descriptions

### Prompt 7 — Data Visualization Recommender

Use this when you have data and need to decide the best way to visualize it on a slide.

```
You are a data visualization expert who designs charts for business presentations. Recommend visualizations for my data.

Data sets I need to visualize:

Data Set 1:
- What it shows: [DESCRIBE — e.g., "revenue by quarter for the last 2 years"]
- Number of data points: [HOW MANY]
- Key insight: [WHAT SHOULD THE AUDIENCE TAKE AWAY]
- Comparison type: [TIME SERIES / PART-TO-WHOLE / COMPARISON / DISTRIBUTION / RELATIONSHIP]

Data Set 2:
- What it shows: [DESCRIBE]
- Number of data points: [HOW MANY]
- Key insight: [TAKEAWAY]
- Comparison type: [TYPE]

Data Set 3:
- What it shows: [DESCRIBE]
- Number of data points: [HOW MANY]
- Key insight: [TAKEAWAY]
- Comparison type: [TYPE]

[Add more as needed]

Audience data literacy: [HIGH — FINANCE/DATA TEAM / MEDIUM — GENERAL BUSINESS / LOW — NON-TECHNICAL]
Presentation tool: [POWERPOINT / GOOGLE SLIDES / KEYNOTE]

For each data set, provide:

1. **Recommended Chart Type** — The best visualization with justification:
   - Why this chart type works for this data
   - When to use alternatives
   - Examples of what it would look like (described in detail)

2. **Chart Design Specifications**
   - Axes: labels, ranges, units
   - Colors: what to highlight, what to mute
   - Annotations: what to call out directly on the chart
   - Gridlines: show or hide
   - Legend: position and necessity

3. **Insight Headline** — The chart title that tells the story:
   - BAD: "Revenue by Quarter"
   - GOOD: "Revenue Doubled After Q2 Price Change"

4. **Simplification Recommendations**
   - What data to include vs. exclude
   - Whether to show actual numbers on the chart
   - Whether to use a simplified version on the slide with detail in the appendix

5. **Accessibility Considerations**
   - Color-blind safe palette
   - Pattern alternatives to color-only encoding
   - Label directly on data points vs. relying on legends

6. **Common Mistakes**
   - 3D charts (never)
   - Dual y-axes (rarely)
   - Pie charts with too many slices (max 5)
   - Starting y-axis at non-zero (when it's deceptive)

The best data visualization makes the insight obvious without the presenter saying a word.
```

## Narrative Arcs & Presentation Storytelling

### Prompt 9 — Presentation Narrative Arc Designer

Use this to design the overall emotional and logical narrative of any presentation.

```
You are a narrative designer who structures presentations as stories. Design the narrative arc for my presentation.

Presentation topic: [TOPIC]
Presentation type: [PITCH / SALES / KEYNOTE / REPORT / TRAINING]
Number of slides: [COUNT]
Duration: [MINUTES]
Audience: [WHO]

Current slide order:
1. [SLIDE TITLE AND BRIEF CONTENT]
2. [SLIDE TITLE]
3. [SLIDE TITLE]
[Continue for all slides]

The transformation I want the audience to undergo:
- Before the presentation, they think: [CURRENT BELIEF/STATE]
- After the presentation, they think: [DESIRED BELIEF/STATE]
- After the presentation, they do: [DESIRED ACTION]

Generate:

1. **Narrative Structure Assessment**
   - Current narrative: what story does the current slide order tell?
   - Strengths: where the story works
   - Gaps: where the story breaks down or feels flat

2. **Recommended Narrative Arc**
   - Choose the best story structure for this presentation type:
     - **The Hero's Journey** (audience is the hero, you're the guide)
     - **Problem-Solution-Proof** (classic sales/pitch)
     - **Situation-Complication-Resolution** (consulting/strategy)
     - **Before-Bridge-After** (transformation story)
     - **And-But-Therefore** (scientific/logical argument)
   - Map each slide to its role in the story

3. **Emotional Journey Map**
   - For each section of the presentation:
     - What the audience should feel (curiosity, concern, excitement, relief, urgency)
     - What creates that emotion (story, data, humor, surprise)
     - Intensity level (1-10)
   - Describe the emotional "shape" of the presentation (like a story arc curve)

4. **Resequencing Recommendation**
   - If the current order needs changing: the recommended new order
   - Why: how the new order strengthens the narrative
   - What to add or remove

5. **Key Narrative Moments**
   - The Hook: what grabs attention in the first 60 seconds
   - The Turn: the pivotal moment where understanding shifts
   - The Climax: the highest-stakes moment
   - The Resolution: the satisfying conclusion
   - The Call to Action: the final push

6. **The One-Sentence Story**
   - Reduce the entire presentation to one sentence:
   - "[AUDIENCE] will discover that [INSIGHT] and decide to [ACTION]"

7. **Transition Sentences**
   - For each slide-to-slide transition: the connecting sentence that maintains narrative flow

The best presentations don't feel like presentations — they feel like stories. This framework should create that experience.
```

### Prompt 10 — Presentation Opening and Closing Writer

Use this when you need a powerful opening and a memorable closing for any presentation.

```
You are a speechwriting expert who crafts memorable openings and closings. Write the opening and closing for my presentation.

Presentation topic: [TOPIC]
Audience: [WHO]
Tone: [INSPIRING / PROVOCATIVE / WARM / URGENT / PROFESSIONAL]
Duration: [TOTAL PRESENTATION LENGTH]
What the presentation covers (briefly): [2-3 SENTENCE SUMMARY]
The one thing the audience should remember: [SINGLE KEY MESSAGE]
Desired audience action: [WHAT YOU WANT THEM TO DO AFTER]

Personal story available (optional): [A BRIEF PERSONAL STORY RELATED TO THE TOPIC]
Relevant data point (optional): [A SURPRISING STATISTIC YOU CAN USE]

Generate:

**OPENINGS — 5 Options (each 45-90 seconds):**

1. **The Bold Statement Opening**
   - Starts with a declarative sentence that challenges assumptions
   - Exact script with delivery notes

2. **The Story Opening**
   - Starts with a personal or relatable story
   - Exact script with emotional beats marked

3. **The Question Opening**
   - Starts with a question that reframes the topic
   - Exact script with pause marks

4. **The Data Opening**
   - Starts with a surprising statistic
   - Exact script with emphasis marks

5. **The Scenario Opening**
   - Starts with "Imagine..." and paints a vivid picture
   - Exact script with sensory details

**For each opening, include:**
- The exact words to say
- What to show on screen (slide content)
- Body language and delivery guidance
- Transition sentence to the first content slide

**CLOSINGS — 5 Options (each 30-60 seconds):**

1. **The Callback Closing**
   - Returns to the opening story/question/statement and completes it
   - Creates a satisfying sense of closure

2. **The Challenge Closing**
   - Issues a direct challenge to the audience
   - "The question isn't whether [X] — it's whether you'll [Y]"

3. **The Future Vision Closing**
   - Paints a picture of what the future looks like if the audience acts
   - Inspiring and optimistic

4. **The Simple Truth Closing**
   - Strips away complexity to deliver one profound, simple statement
   - Quiet, powerful, resonant

5. **The Quotable Closing**
   - A single line designed to be remembered, shared, and quoted
   - Under 15 words

**For each closing, include:**
- The exact words to say
- Delivery guidance (pace, volume, eye contact)
- What to show on the final slide
- Whether to pause before or after the final line

The opening earns the right to be heard. The closing earns the right to be remembered.
```

### Prompt 11 — Analogy and Metaphor Generator for Presentations

Use this when you need to explain something complex and want a metaphor or analogy that makes it instantly understandable.

```
You are a communication expert who uses analogies and metaphors to make complex ideas accessible. Generate analogies for my presentation.

I need to explain: [THE CONCEPT — e.g., "how our machine learning model improves with more data," "why our pricing has multiple tiers," "how our security architecture protects customer data"]
To this audience: [WHO — THEIR TECHNICAL LEVEL AND CONTEXT]
In this presentation: [TYPE — PITCH / SALES / KEYNOTE / TRAINING]

   - Images that would reinforce each analogy on a slide
   - Icons or illustrations to consider
   - Before/after visual concepts

## Audience-Specific Customization

### Prompt 12 — Presentation Audience Adapter

Use this to adapt an existing presentation for a different audience without rebuilding it from scratch.

```
You are a communication strategist who adapts presentations for different audiences. Help me customize my presentation.

My current presentation is designed for: [ORIGINAL AUDIENCE — e.g., "technical engineers," "executive leadership," "sales team"]
I need to adapt it for: [NEW AUDIENCE — e.g., "non-technical board members," "customer success team," "new hires"]

Current presentation summary:
- Topic: [TOPIC]
- Number of slides: [COUNT]
- Key messages:
  1. [MESSAGE 1]
  2. [MESSAGE 2]
  3. [MESSAGE 3]
- Slide-by-slide overview:
  [LIST EACH SLIDE TITLE AND BRIEF CONTENT]

What the new audience cares about: [THEIR PRIORITIES AND CONCERNS]
What the new audience's knowledge level is: [WHAT THEY KNOW AND DON'T KNOW ABOUT THE TOPIC]
Time available: [SAME AS ORIGINAL / SHORTER / LONGER]

Generate:

1. **Audience Gap Analysis**
   - What the original audience knows that the new audience doesn't
   - What the new audience cares about that the original didn't
   - Jargon or terminology to change
   - Examples that need to be swapped for more relevant ones

2. **Slide-by-Slide Adaptation Guide**
   For each slide:
   - Keep / Modify / Remove / Add
   - If Modify: what specifically changes (content, framing, depth, examples)
   - If Remove: why it doesn't serve this audience
   - If Add: what new slide is needed and where

3. **Language Adjustments**
   - Words to change (e.g., "API integration" to "connecting your systems")
   - Tone shifts (e.g., more formal, more casual, more empathetic)
   - Acronyms to spell out or explain

4. **New Examples and Stories**
   - Replace original examples with ones relevant to the new audience
   - New analogies that resonate with their experience

5. **Emphasis Shifts**
   - What to spend more time on (for this audience)
   - What to speed through (they either know it or don't need it)
   - New questions to anticipate from this audience

6. **Adapted Speaker Notes**
   - Key adjustments to talking track for each modified slide

The best presenters don't give one presentation to every audience — they give the right presentation to each audience.
```

### Prompt 13 — Executive Summary Slide Generator

Use this to create a single executive summary slide that captures an entire presentation for a time-pressed executive.

```
You are an executive communication specialist. Create an executive summary slide for my presentation.

My full presentation covers:
- Topic: [TOPIC]
- Total slides: [COUNT]
- Key data points: [LIST 3-5 MOST IMPORTANT NUMBERS]
- Key findings: [LIST 3-5 KEY FINDINGS OR CONCLUSIONS]
- Recommendation: [YOUR MAIN RECOMMENDATION]
- Ask: [WHAT YOU NEED FROM THE EXECUTIVE]

The executive has: [2 MINUTES / 5 MINUTES / WILL ONLY READ THE SLIDE — WON'T HEAR A PRESENTATION]
Executive's role: [CEO / CFO / CTO / BOARD MEMBER / VP OF __]
What they care most about: [REVENUE / COST / RISK / GROWTH / CUSTOMERS / TEAM]

Generate:

1. **The One-Slide Summary** — Everything critical on a single slide:
   - Headline: the single most important takeaway (under 10 words)
   - Situation: 1-2 sentences (what's happening)
   - Key Data: 3-4 numbers in large format
   - Recommendation: 1 sentence (what to do)
   - Ask: 1 sentence (what you need from them)
   - Layout description for visual hierarchy

2. **The Three-Sentence Version** — For the executive who gets the slide in an email:
   - Sentence 1: Here's what's happening
   - Sentence 2: Here's what it means
   - Sentence 3: Here's what we should do

3. **The Verbal Pitch** — 60-second version for an elevator or hallway conversation:
   - Exact words, designed for speaking
   - Gets to the point in 15 seconds, uses remaining time for evidence

4. **Pre-Read Email** — An email to send before the meeting with the executive:
   - Subject line
   - 5-line email body
   - Attachment reference ("detailed analysis attached")

5. **Decision-Ready Format** — If the executive needs to make a decision:
   - Decision needed: [CLEAR STATEMENT]
   - Option A: [DESCRIPTION + PROS + CONS]
   - Option B: [DESCRIPTION + PROS + CONS]
   - Recommendation: [WHICH OPTION AND WHY]
   - Deadline: [WHEN THE DECISION IS NEEDED]

Executives process information in inverted pyramid format — lead with the conclusion, then support it with evidence. Never make them wait for the punchline.
